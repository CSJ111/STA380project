functions.R:

You use \dontrun{} but in all honesty I would get rid of that.
In particular, something you can do that's quite nice in R for a package is run example().
This can be very useful for debugging or testing something quickly.

Example (this is for a package I work on)
library(lme4)
example(lmer)
This will generate a linear mixed model (something beyond the scope of undergrad) and you can view and experiments with the plots.

So, being able to support example( function_name ) is generally useful.
